========================================================================
       WIN32 APPLICATION : rerun
========================================================================

reRun - переадресовывает вызов программы

Иногда требуется переадресовать вызов программы на другое месторасположение
или файл. Например, в каком-то приложении жёстко прописан путь к другому
вызываемому приложению, которое у вас установлено немного не по тому пути,
который зашит в программу.
Для этого и пригодится reRun. Скопируете его в "неправильное" место и
в конфигурационном файле укажите "правильный" путь к приложению.

redir_to_1="\"%sysdir%\\calc.exe\" %cmdln%"
redir_to_2="запусить ещё одно приложение"

%here%       - текущая директория
%cmdln%      - параметры командной строки
%windir%     - директория Windows
%sysdir%     - путь к Windows\System32
%progdir%    - путь к Program Files
%deskdir%    - путь к "Рабочий стол" пользователя
%mydocdir%   - директория "Мои документы"
%profdir%    - путь к профайлу пользователя
%startdir%   - путь к папке "Автозапуск"
%%           - символ "процента"

options="6"  - показать сообщение при ошибке запуска и первать выполнение

Флаги options:
0x00000001 - показать параметры запуска (для отладки)
0x00000002 - показывать сообщение при ошибке запуска
0x00000004 - прервать запуск приложений при ошибке запуска предыдущего

7 = 0x01 | 0x02 | 0x04
6 =        0x02 | 0x04
3 = 0x01 | 0x02

Конфигурационный файл должен иметь тоже имя, что и испоняемый (.exe)
